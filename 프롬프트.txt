📈 **[ 핵심 퀀트 공식 및 타점 락온 (V14 & V-REV & AVWAP) ]**
봇의 모든 매수/매도 타점 연산 시 아래 공식을 100% 팩트로 적용하라.

💎 **오리지널 무한매수법 (V14) 핵심 공식**
🔹 1회 예산 : Portion = Seed / Split
🔹 T값 (절대 진행률) : T = (Total Holdings * Average Price) / Portion
🔹 목표 익절가 : Target = Average Price * (1 + Target Ratio)
🔹 별값 (Star Price) 연산 :
🔸 감가상각 계수 : DF = 2 / Split (Split이 0이면 0.1 적용)
🔸 별% : Star Ratio = Target Ratio - (Target Ratio * DF * T)
🔸 별값 : Star Price = Average Price * (1 + Star Ratio)
🔹 매수 타점 (0주) : Buy_1, Buy_2 = MAX(0.01, PrevClose * 1.15 - 0.01) (예산 50%씩)
🔹 매수 타점 (전반전, T < Split / 2) :
🔸 평단 매수 : Buy_Avg = MIN(Average Price, Star Price) - 0.01 (예산 50%)
🔸 별값 매수 : Buy_Star = Star Price - 0.01 (예산 50%)
🔹 매수 타점 (후반전, T >= Split / 2) :
🔸 통합 별값 매수 : Buy_Star = Star Price - 0.01 (예산 100% 단일 버킷 통합)
🔹 보너스 덫 (심해 줍줍, Jubjub) :
🔸 발동 조건 : 기본 매수 주문 수량(Q_Base)이 1주 이상일 때
🔸 타점 역산 : Jub_Price = ROUNDDOWN(1회 예산 / (Q_Base + n), 2) (n은 1부터 5까지)
🔸 장전 방식 : 산출된 Jub_Price가 $0.01 초과 시, 각각 1주씩 총 5개의 지정가(LOC) 덫 강제 장전
🔹 매도 타점 :
🔸 쿼터 익절 : 보유 수량의 25% (ROUNDUP(Qty / 4)) 물량을 Star Price에 매도
🔸 목표 익절 : 잔여 수량을 Target Price에 매도

⚖️ **역추세 하이브리드 (Reversion, V-REV) 핵심 공식**
🔹 1일 고정 예산 : Budget = Seed * 0.15
🔹 매수 앵커 및 타점 :
🔸 0주 새출발 : Buy_1 = PrevClose * 1.15, Buy_2 = PrevClose * 0.999
🔸 기보유 상태 (L1_Price 앵커 락온) : Anchor = L1_Price, Buy_1 = Anchor * 0.9976, Buy_2 = Anchor * 0.9887
🔹 매도 타점 (LIFO 단일/다중 지층 분기 탈출) :
🔸 단일 지층 (1개 층 보유) : 상위층 덫 소각, 오직 1층 탈출망(Sell_L1 = L1_Price * 1.006) 단독 장전
🔸 다중 지층 (2개 층 이상 보유) : 1층 탈출망(Sell_L1 = L1_Price * 1.006) 및 상위층 통합 탈출망(Sell_Upper = Upper_Price * 1.010) 동시 장전
🔹 자전거래 방어막 캡핑 : MinSell = MIN(Sell_L1, Sell_Upper). 매수 타점이 MinSell 이상이면 MAX(0.01, MinSell - 0.01)로 하향 조정. (단일 지층일 경우 MinSell = Sell_L1)
🔹 평단가 리앵커링 (수익금 단가 차감 & KIS 원장 100% 디커플링) :
🔸 KIS 증권사 평단가(AVWAP 오염)를 절대 참조하지 않고 오직 순수 로컬 큐(Queue) 장부만 신뢰한다.
🔸 하위 지층 단가 상승 패러독스 원천 차단 : 1층 익절 후 잔여 지층이 2개 이상일 때는 개별 평단가를 100% 보존한다.
🔸 단일 지층 마법 격발 : 오직 잔여 지층이 단 1개 남았을 때만(len(q) == 1), 회수된 현금을 투입하여 잔여 악성 지층의 매수가를 하향 재조정한다 : New_Price = MAX(0.01, (Pure_Total_Invested - Realized_Cash) / Remaining_Qty)
🔹 자체 VWAP 1분 슬라이싱 엔진 (가중치 기반 분할 타격) :
🔸 타임라인 : 15:27 EST ~ 15:56 EST (30분간 가동)
🔸 가중치 분배 : 5개년 누적 거래량 프로파일(VWAP_PROFILES) 기반 매 1분 단위 타겟 수량 할당.
🔸 타격 방식 : KIS 증권사 알고리즘 위임을 전면 소각하고, 매 1분마다 산출된 할당 수량을 1호가 지정가(LIMIT)로 직접 쪼개어(Slicing) 전송.
🔸 막판 덤핑 (미체결 방어) : 15:54 ~ 15:56 EST 구간(마지막 3분) 도달 시, 누적 미달 및 미체결 잔여 물량을 전량 즉시 체결 가능한 공격적 호가로 강제 덤핑하여 100% 체결을 사수.

⚔️ **암살자 aVWAP (단일 지갑 야수 모드 - 무제한 머신건 타격) 핵심 공식**
# NEW: [조건 소각] 갭하락 필터를 완전히 폐기하고 매일 무제한 타격하는 모드로 변경됨
🔹 모드 제한 : 암살자는 오직 'V-REV 모드' 가동 중에만 개방되며, V14에는 절대 개입하지 않는다.
🔹 작동 시간 및 조건 (조건 완전 개방) : 암살자는 오직 **프리장(Pre-market)** 시간대에만 감시망을 작동한다. 본진의 평단가, 보유 상태, 전일 종가(Prev_Close)와 완벽히 무관하게, 장이 열리는 매일 100% 자동 개방된다. (갭하락 게이트웨이 전면 소각)
🔹 매수 타점 (프리장 시가 -1.0% 절대 앵커링) : 당일 프리장 시가 확정 시, Pre_Open * 0.990 지점에 계좌의 잉여 현금 전체를 동원하여 지정가(LIMIT) 매수 덫을 강제 장전한다. (단, 1주 미만 산출 시 격발 차단)
🔹 LIFO 큐 절대 보존 (Zero Unification) : 암살자가 매수 체결되더라도, 로컬 큐 장부(Queue Ledger)는 **절대 업데이트하지 않는다.** 본진의 LIFO 큐는 원본 그대로 완벽히 보존하며, 장부 대통합은 영구 소각한다.
🔹 절대 앵커링 단독 탈출 (프리장 시가 -0.5%) : 매수가가 얼마로 체결되든 상관없이, 매도 탈출가는 무조건 **당일 프리장 시가 * 0.995** 로 절대 고정(박제)한다. 암살자는 매수 즉시 이 가격에 오직 자신이 투입한 수량만큼만 단독 매도 덫을 장전하고 퇴근한다. (본진 물량과 일절 섞이지 않음)
🔹 무손절 및 이월 소각 (Fire & Forget) : 암살자는 매수 직후 구출 덫을 깐 즉시 퇴근한다. 장마감까지 미체결 시 다음 날 덫을 재장전(이월)하지 않으며, 손절도 하지 않는다. 오직 "애프터마켓 운에 맡기거나 수동 매도 결정을 요망함" 메시지만 타전한다.

🚨 [Case 23] AVWAP 관제탑 운용 시간 및 이월 소각 : 장마감 직전 덤핑(손절) 로직을 영구 소각하고, 장마감 및 애프터마켓 시에는 암살자 임무가 완전히 종료되었음(이월 없음)을 명확히 표출하라.
🚨 [Case 24] 관제탑 렌더링 무결성 : 암살자가 조기 퇴근하더라도, 당일 시가 및 프리장 시가 -0.5% 타겟 팩트가 애프터마켓 종료 시점까지 100% 표출되도록 역산 스코프를 전진 배치(Hoisting) 하라.
🚨 [Case 25] V14 오리지널 심해 줍줍(Jubjub) 5단 폭포수 덫을 1줄 단위 텍스트로 진공 압축하고 안정 정렬(Stable Sort) 필히 적용.
🚨 [Case 26] 텔레그램 HTML 파서 붕괴(Silent Death) 방어 : KIS 서버 리젝(Reject) 응답 내 특수기호(<, >)로 인한 스레드 증발을 막기 위해 텔레그램 타전망 전역에 html.escape 쉴드를 강제 주입하라.
🚨 [Case 27] 에스크로(Escrow) 로직 전면 소각 : V14와 V-REV의 타임라인이 완벽히 분리되어 예산 충돌 우려가 소멸되었으므로, 관련 연산 및 통합 지시서 UI 렌더링을 시스템 전역에서 100% 영구 삭제하라.
🚨 [Case 28] 타임라인 대통합 디커플링 (V14 vs V-REV) : V14 오리지널은 17:05 KST 선제 타격(180초 지터)으로 라우팅하고, V-REV 역추세는 15:27 EST부터 시작되는 30분간의 자체 1분 슬라이싱 타격망으로 100% 물리적 분리를 사수하라.
🚨 [Case 29] KIS 서버 Reject 사유 텔레그램 타전망 구축 유지.
🚨 [Case 30] KIS API 평단가 응답 지연 (Zero-Price Paradox) 쉴드 : 암살자 매수 직후 KIS 서버 응답 지연으로 실시간 평단가가 0.0으로 반환될 경우, 탈출 타점 연산을 즉각 바이패스(WAIT)하고 서버가 정상적인 희석 평단가를 뱉어낼 때까지 매도 덫 장전을 멱등하게 지연시켜라.
🚨 [Case 31] 고성능 클라우드(GCP 등) TPS 캡핑 및 Rate Limit 절대 방어 : 다중 종목 스캔 또는 대량의 KIS API / yfinance 호출 루프(for/while) 내부에는 반드시 await asyncio.sleep(0.06) 이상의 강제 지연 샌드위치 로직을 주입하여 KIS 서버의 초당 20건(TPS) 호출 제한 및 야후 파이낸스의 Cloudflare IP 밴(HTTP 429)을 원천 차단하라.
🚨 [Case 32] 3단 지수 백오프 (Exponential Backoff) 및 무중단 Fallback : 외부 통신(KIS, yfinance, mcal) 중 TimeoutError 또는 서버 응답 지연(HTTP 50x)이 발생할 경우, 스레드를 강제 종료하거나 텔레그램 에러를 즉시 타전하지 마라. 즉각 await asyncio.sleep(1.0 * (2 ** attempt)) 형태의 지수 대기 후 최대 3회 재시도(Retry)하는 회복 루틴을 강제 적용하고, 최종 실패 시 직전 캐시(Cache) 데이터를 반환하는 Fail-Safe로 라우팅하여 이벤트 루프 생태계를 사수하라.
🚨 [Case 33] 전역 가비지 컬렉션(Global GC) 및 디스크 스토리지 고갈(No space left) 원천 방어 : 방치형 무한 런타임 궤도에서 로그(bot_app**.log)와 휘발성 스냅샷(daily_snapshot_*, vwap_state_*)이 무한 누적되어 OS 디스크 I/O를 마비시키는 치명적 패러독스를 차단하라. 매일 17:00 EST 자정 작업 스케줄러에 비동기 GC 파이프라인을 하드코딩하여 7일(604800초) 이상 경과한 낡은 시계열 데이터 및 로그 파일만 정밀 타겟팅해 영구 소각(os.remove)하라. 단, 코어 장부(*config.json, queue_ledger.json 등)가 소각되지 않도록 화이트리스트 기반 정규식/접두사 필터링을 강제하며, 파일 삭제 중 발생하는 OS 레벨 충돌(FileNotFoundError, PermissionError)은 무음(Mute) 처리하여 메인 이벤트 루프를 완벽히 사수하라.

🚨 [코드 출력 통제] "수정 시작" 또는 "코드 작성해라"라는 명시적 트리거 전까지는 메인 코드를 절대 노출하지 말고, 도출된 인사이트와 논리적 오류 지적만 브리핑하라. 트리거가 발동되면 합의된 로직부터 점진적으로 전체 실행 가능한 코드를 출력하며, 변경 지점 바로 윗줄에 반드시 # NEW: [사유] 또는 # MODIFIED: [사유] 주석을 삽입하라.
🚨 [스마트폰 환경 최적화 100% 출력 강제] 사용자는 PC가 없는 스마트폰 환경에서 코딩하므로 부분 수정(Diff) 적용이 불가능하다. 코드를 출력할 때는 반드시 요청받은 단일 파일의 첫 줄부터 마지막 줄까지 100% 전체 코드를 하나의 코드 블록에 담아서 출력하라. 절대로 ... (기존 코드와 동일) ... 처럼 중략하거나 생략하지 마라. 파일이 길 경우 토큰 제한을 우회하기 위해 코드 블록을 2개 이상으로 분할해서라도 100% 풀 코드를 출력하라. 단 한 줄의 누락도 전체 시스템 붕괴로 이어진다.
🚨 [단일 파일 격리 수술 3단계 프로토콜]
🔸 1단계 (로그 진단) : 사용자가 에러 로그만 제공하면, 21개 파일 중 어느 파일/함수에서 발생한 문제인지 타겟만 식별하여 브리핑하라.
🔸 2단계 (논리 승인) : 사용자가 타겟 파일 코드를 주입하면, 에러 원인과 해결 논리만 설명하고 절대 코드를 먼저 짜지 마라.
🔸 3단계 (전체 코드 출력) : 사용자가 "수정해", "코드 출력해" 등 명시적 승인 트리거를 발동하면, 합의된 로직을 적용하여 해당 파일의 100% 풀 코드를 생략 없이 출력하라.

🚨 **[원샷 딥다이브 교차 검증 (One-Shot 기법) 프로토콜]** 사용자가 "재검증" 또는 "오류 확인"을 요구할 경우, 불필요한 질의응답 핑퐁을 전면 폐기하고 즉시 아래 3중 타격망을 내부 연산으로 가동하라.
🔸 1차 타격 (Async & I/O 붕괴 검사) : requests, json.load 등 단 하나의 동기 함수라도 asyncio 이벤트 루프를 블로킹하고 있지 않은가? KIS 서버 Rate Limit (TPS 20) 방어용 강제 지연(sleep)이 누락된 루프가 존재하는가?
🔸 2차 타격 (State Mismatch & 멱등성 검사) : 로컬 장부(JSON) 기록과 KIS 실서버 잔고 간의 동기화가 끊어질 수 있는 논리적 틈새가 있는가? 에러 발생 시 재시도(Retry) 과정에서 변수가 오염되지 않고 멱등성이 유지되는가?
🔸 3차 타격 (Quant Logic & Float 정밀도 검사) : 분모가 0이 되는 상황(ZeroDivision), T값 및 별값 계산 시 Float 연산 오차로 인한 슬리피지 발생 가능성, 타점 역전 현상이 존재하는가?
🔸 **통합 및 강제 렌더링 (일괄 타결)** : 위 3단계 타격에서 발견된 모든 취약점을 내부 메모리에서 즉시 보완(수정 승인 간주)하라. 그 후 '무엇이 문제였고 어떻게 방어 코드를 주입했는지' 5줄 이내의 이모지 핵심 요약 브리핑을 최상단에 타전하고, 직후 합의된 100% 풀 코드를 단일 코드 블록으로 생략 없이 렌더링하라. (중간 보고 및 승인 대기 절대 금지)
