📈 **[ 핵심 퀀트 공식 및 타점 락온 (V14 & V-REV & AVWAP) ]**
봇의 모든 매수/매도 타점 연산 시 아래 공식을 100% 팩트로 적용하라.
💎 **오리지널 무한매수법 (V14) 핵심 공식**
🔹 1회 예산 : Portion = Seed / Split
🔹 T값 (절대 진행률) : T = (Total Holdings * Average Price) / Portion
🔹 목표 익절가 : Target = Average Price * (1 + Target Ratio)
🔹 별값 (Star Price) 연산 :
🔸 감가상각 계수 : DF = 2 / Split (Split이 0이면 0.1 적용)
🔸 별% : Star Ratio = Target Ratio - (Target Ratio * DF * T)
🔸 별값 : Star Price = Average Price * (1 + Star Ratio)
🔹 매수 타점 (0주) : Buy_1, Buy_2 = MAX(0.01, PrevClose * 1.15 - 0.01) (예산 50%씩)
🔹 매수 타점 (전반전, T < Split / 2) :
🔸 평단 매수 : Buy_Avg = MIN(Average Price, Star Price) - 0.01 (예산 50%)
🔸 별값 매수 : Buy_Star = Star Price - 0.01 (예산 50%)
🔹 매수 타점 (후반전, T >= Split / 2) :
🔸 통합 별값 매수 : Buy_Star = Star Price - 0.01 (예산 100% 단일 버킷 통합)
🔹 보너스 덫 (심해 줍줍, Jubjub) :
🔸 발동 조건 : 기본 매수 주문 수량(Q_Base)이 1주 이상일 때
🔸 타점 역산 : Jub_Price = ROUNDDOWN(1회 예산 / (Q_Base + n), 2) (n은 1부터 5까지)
🔸 장전 방식 : 산출된 Jub_Price가 $0.01 초과 시, 각각 1주씩 총 5개의 지정가(LOC) 덫 강제 장전
🔹 매도 타점 :
🔸 쿼터 익절 : 보유 수량의 25% (ROUNDUP(Qty / 4)) 물량을 Star Price에 매도
🔸 목표 익절 : 잔여 수량을 Target Price에 매도
⚖️ **역추세 하이브리드 (Reversion, V-REV) 핵심 공식**
🔹 1일 고정 예산 : Budget = Seed * 0.15
🔹 매수 앵커 및 타점 :
🔸 0주 새출발 : Buy_1 = PrevClose * 1.15, Buy_2 = PrevClose * 0.999
🔸 기보유 상태 (L1_Price 앵커 락온) : Anchor = L1_Price, Buy_1 = Anchor * 0.9976, Buy_2 = Anchor * 0.9887
🔹 매도 타점 (LIFO 탈출) :
🔸 1층(L1) 탈출 : Sell_L1 = L1_Price * 1.006
🔸 상위층(Upper) 탈출 : Sell_Upper = Upper_Price * 1.010
🔹 자전거래 방어막 캡핑 : MinSell = MIN(Sell_L1, Sell_Upper). 매수 타점이 MinSell 이상이면 MAX(0.01, MinSell - 0.01)로 하향 조정.
⚔️ **암살자 aVWAP (다중 출격 최적화) 핵심 공식**
🔹 앵커 타점 및 오프셋 (MA5 Base Offset) : Offset = MA5(5일 평균 종가) * Amp_5d * 0.45 (V자 반등장에서는 얕은 눌림목 타격, 폭락장에서는 심해 방어막으로 자동 가변 전개됨. 단, MA5 결측 시 PrevClose로 Safe Fallback 락온)
🔹 매수 앵커 및 타점 : T_H = PM_H (프리장 고가) - Offset
🔹 매도 타점 (익절가) : Target = Average Price * 1.02 (2.0% 타겟 익절)
🛡️ **[ 34대 치명적 엣지 케이스 절대 방어망 ]**

🚨 [Case 11] AVWAP 듀얼 모드 (단일/다중 출격) 및 조건부 타임슬라이싱 기요틴 셧다운 락온 : 사용자가 텔레그램 UI를 통해 암살자 작전 모드를 선택할 수 있게 하라.
🔸 단일 타격 (Single-Sortie) : 프리장(04:00~09:29) 선제 매수 체결 후 2.0% 익절 시 즉각 당일 영구 동결(SHUTDOWN).
🔸 다중 출격 (Multi-Sortie) : 프리장에 1차 매수가 체결된 날에 한해, 익절 후 T_H 재도달 시 15:20 전까지 무한 재매수/재익절(다중 출장) 파이프라인 개방.
🔸 조건부 기요틴 (공통) : 모드와 상관없이 09:30 EST 정각까지 1차 매수가 체결되지 않으면, 정규장 휩소 방어를 위해 무조건 당일 영구 동결(SHUTDOWN)을 격발하라.
🔸 타임 슬라이싱 차단 (Time-Slicing) : 13:00 EST 도달 시 신규 지정가 덫 장전을 원천 차단(SHUTDOWN)하되, 기보유 물량에 대한 +2.0% 익절 감시 및 관제탑 레이더 표출은 15:20 덤핑 시각 및 장마감까지 100% 디커플링하여 사수하라.

🚨 [Case 18] 로컬 예약 스냅샷 전면 폐기 및 KIS 원장(order_resv_list) 직접 연동 기반 취소 집행.
🚨 [Case 19] Reversion 모드 기보유 시 매수 앵커를 최근 1지층 평단가(l1_price)로 강제 락온.
🚨 [Case 20] KIS 알고리즘 VWAP 지터 동적 시프트(max(15:26:00 EST, 현재시각+3분)) 및 KST 래핑 락온.
🚨 [Case 21] V14 후반전 별값 매수 예산 50:50 기계적 분할 금지 및 100% 단일 버킷 통합 타격.
🚨 [Case 22] +0.2% 슬리피지 가산 로직 전면 소각 및 실 실시간 매도 1호가 직결 요격 롤백.
🚨 [Case 23] AVWAP 관제탑 운용 시간을 정규장 마감(16:00 EST)까지 연장하고, 상태 마스킹(동결) 해제.
🚨 [Case 24] 관제탑 렌더링 무결성 : 암살자가 조기 퇴근하더라도, 프리장 및 정규장 고가/저가(PM_H, T_H 등) 팩트가 애프터마켓 종료 시점까지 100% 표출되도록 역산 스코프를 전진 배치(Hoisting) 하라.
🚨 [Case 25] V14 오리지널 심해 줍줍(Jubjub) 5단 폭포수 덫을 1줄 단위 텍스트로 진공 압축하고 안정 정렬(Stable Sort) 필히 적용.
🚨 [Case 26] 텔레그램 HTML 파서 붕괴(Silent Death) 방어 : KIS 서버 리젝(Reject) 응답 내 특수기호(<, >)로 인한 스레드 증발을 막기 위해 텔레그램 타전망 전역에 html.escape 쉴드를 강제 주입하라.
🚨 [Case 27] 에스크로(Escrow) 로직 전면 소각 : V14와 V-REV의 타임라인이 완벽히 분리되어 예산 충돌 우려가 소멸되었으므로, 관련 연산 및 통합 지시서 UI 렌더링을 시스템 전역에서 100% 영구 삭제하라.
🚨 [Case 28] 암살자 수동 요격(Manual Fire) 동적 스위칭 제어 : 팻핑거 방어를 위해 관제탑 수동 제어 버튼은 프리장/정규장(PRE/REG)에만 노출하라. 수동 요격 시 현재가 제한을 전면 해제하되, 타점은 무조건 T_H 순수 지정가(LIMIT)로 락온하여 KIS 호가창에 덫을 장전하라. 덫이 장전(미체결)된 상태에서는 관제탑의 '수동 요격' 버튼을 '수동 매수취소 (Nuke Trap)' 버튼으로 동적 스위칭하고, 1분봉 감시망이 체결을 확정 지으면 취소 버튼을 소각하여 논리적 충돌을 원천 차단하라.
🚨 [Case 29] 타임라인 대통합 디커플링 (V14 vs V-REV) : V14 오리지널은 17:05 KST 선제 타격(180초 지터)으로 라우팅하고, V-REV 역추세는 기존 15:26 EST 지연 투하망을 유지하여 100% 물리적 분리를 사수하라.
🚨 [Case 30] KIS 서버 Reject 사유 텔레그램 타전망 구축 유지.
🚨 [Case 31] AVWAP 1분봉 시차 패러독스(Time-Shield Decoupling) 원천 차단 : 지정가 매수 덫 장전(PLACE_TRAP) 시 기준 캔들의 시각(trap_placed_time)을 상태 캐시에 락온하라. 장전 직후 1분 이내에 유입되는 캔들의 저가(Low)가 T_H를 관통하더라도, 주문 전송 지연(Latency) 중 발생한 노이즈이므로 체결 검증(VERIFY_TRAP_FILL)을 강제 바이패스하고 멱등하게 대기(TRAP_WAIT)시켜 덫 자폭(Self-Destruct)을 방어하라.
🚨 [Case 32] 고성능 클라우드(GCP 등) TPS 캡핑 및 Rate Limit 절대 방어 : 다중 종목 스캔 또는 대량의 KIS API / yfinance 호출 루프(for/while) 내부에는 반드시 await asyncio.sleep(0.06) 이상의 강제 지연 샌드위치 로직을 주입하여 KIS 서버의 초당 20건(TPS) 호출 제한 및 야후 파이낸스의 Cloudflare IP 밴(HTTP 429)을 원천 차단하라.
🚨 [Case 33] 3단 지수 백오프 (Exponential Backoff) 및 무중단 Fallback : 외부 통신(KIS, yfinance, mcal) 중 TimeoutError 또는 서버 응답 지연(HTTP 50x)이 발생할 경우, 스레드를 강제 종료하거나 텔레그램 에러를 즉시 타전하지 마라. 즉각 await asyncio.sleep(1.0 * (2 ** attempt)) 형태의 지수 대기 후 최대 3회 재시도(Retry)하는 회복 루틴을 강제 적용하고, 최종 실패 시 직전 캐시(Cache) 데이터를 반환하는 Fail-Safe로 라우팅하여 이벤트 루프 생태계를 사수하라.
🚨 [Case 34] 전역 가비지 컬렉션(Global GC) 및 디스크 스토리지 고갈(No space left) 원천 방어 : 방치형 무한 런타임 궤도에서 로그(bot_app**.log)와 휘발성 스냅샷(daily_snapshot_*, vwap_state_*)이 무한 누적되어 OS 디스크 I/O를 마비시키는 치명적 패러독스를 차단하라. 매일 17:00 EST 자정 작업 스케줄러에 비동기 GC 파이프라인을 하드코딩하여 7일(604800초) 이상 경과한 낡은 시계열 데이터 및 로그 파일만 정밀 타겟팅해 영구 소각(os.remove)하라. 단, 코어 장부(*config.json, queue_ledger.json 등)가 소각되지 않도록 화이트리스트 기반 정규식/접두사 필터링을 강제하며, 파일 삭제 중 발생하는 OS 레벨 충돌(FileNotFoundError, PermissionError)은 무음(Mute) 처리하여 메인 이벤트 루프를 완벽히 사수하라.
